# 引継書 — モンスト ノマクエ放置周回 bot

このドキュメントを引き継ぐ Claude Code へ。会話文脈は無い前提で、これ単独で自走できるよう書いてある。`monst_rank_bot.py` が現状の実装。これを「数時間〜半日、無人で安定周回し続ける」状態まで仕上げてほしい。

---

## 1. ゴール

- モンスト（モンスターストライク / mixi）の **ノーマルクエスト（ノマクエ）を実機で自動周回**し、ランク経験値を放置で稼ぐ。
- **「完全カンスト」はゴールにしない**（理由は §2）。目標は「手を触れずにランクを淡々と上げ続けるbotが落ちずに回ること」。

## 2. 重要な前提・制約（設計に直結。必ず守ること）

1. **モンストはフリック（弾く）ゲー。** 単純タップではなく、自陣ユニットからスワイプして発射する物理アクション。バトル中の操作は `adb shell input swipe`（フリック）で行う。ノマクエの雑魚は精密エイム不要なので、**ランダム方向フリック連打**で十分クリアできる。座標決め打ちでの狙撃はしない（脆い）。
2. **実機 Pixel 8a を ADB（root不要）で操作。** エミュレータは使わない（モンストは検出して弾く/凍結リスク）。root化・メモリ改変もしない。入力はADBの正規経路（input tap / swipe）のみ。
3. **スタミナがスループットの上限。** 経験値はスタミナ消費に比例し、回復速度は固定。bot速度を上げても1日の獲得経験値は増えない。参考: ランク2000→2500は1ランクあたり約1300万EXP、普通にプレイすると満ランクまで何十年級。だから完全カンストは非現実的＝「スタミナ切れたら回復待ち→自動再開」のサイクルが落ちないことが本質。
4. **BANリスクは高め。捨て垢で運用。** 2026年3月にモンスト運営が「ゲームプレイ履歴を自動取得できる」よう規約改定済み＝自動操作の監視に積極的。検出されにくさのため、ADB正規入力＋人間っぽい遅延・座標ジッターは必ず維持する。高速化のために挙動を機械的にしすぎない。
5. **判定は画像認識（OpenCV テンプレマッチ）。** 座標は解像度依存なので決め打ちは最小限にし、ボタンはテンプレで探す。

## 3. 現状の実装（`monst_rank_bot.py`）

root不要のADB操作 + OpenCVテンプレマッチによる状態機械。主要パーツ:

- `adb_base()` / `screencap()` / `tap()` / `swipe()` — ADBラッパ。`screencap` は `adb exec-out screencap -p` をBGR ndarrayで返す。`tap` は座標ジッター入り。
- `flick()` — `FLICK_PIVOT` を起点にランダム方向・ランダム振り幅でフリック。
- `match(name, screen)` — `TEMPLATES[name]`（画像＋しきい値）を画面上で探し、中心座標 or None。
- `do_battle()` — `result`/`home` テンプレが出るまでフリック連打。`BATTLE_TIMEOUT` で保険。
- `advance_until(target, taps, point)` — 演出/ダイアログを `advance` 座標タップで送りつつ目的画面まで前進。
- `run()` — メインループ。スタミナ切れ→待機 / リザルト→もう一度→出撃→バトル / 未知画面→advanceタップ、の状態分岐。
- `shot()` — 現在画面を `screen.png` に保存（テンプレ切り出し用）。CLI: `python monst_rank_bot.py [shot|run]`。
- `CONFIG` ブロック — `ADB_SERIAL` / `TEMPLATES` / `COORDS` / `FLICK_*` / タイミング各種。**ここがキャリブレーション対象。**

`templates/` に判定用画像を置く設計（未作成。要採取）。

## 4. 環境セットアップ

```bash
pip install opencv-python numpy
# Pixel 8a: 開発者オプション→USBデバッグON、USB接続
adb devices                     # デバイスが出ればOK
adb shell settings put global stay_on_while_plugged_in 3   # 給電中スリープ防止
```

複数端末接続時のみ `ADB_SERIAL` 環境変数でシリアル指定。

## 5. 残タスク（優先順）

### P0（これが無いと動かない）
1. **テンプレ採取と `templates/` 整備。** `python monst_rank_bot.py shot` で各画面を撮り、ボタンを切り出して `result.png` / `again.png` / `start.png` / `stamina.png` / `home.png` を作る。判定が安定する固有要素を選ぶこと。
2. **`CONFIG` 座標キャリブレーション。** `screen.png` の実解像度に合わせ `FLICK_PIVOT`（自陣中央付近）・`COORDS`（advance/again/start）・`FLICK_RADIUS_*` を調整。マッチしきい値も実測で 0.80〜0.90 に詰める。

### P1（安定周回に必須）
3. **`advance_until` の演出送りロバスト化。** ランクアップ演出・クリア演出・ボーナスステージ開始/結果・各種お知らせダイアログを取りこぼさず送る。実際に回してログを見て、止まった画面のテンプレ/座標を足していくのが確実。
4. **スタミナ切れ判定とリカバリ。** ダイアログ検知→閉じる→`STAMINA_WAIT` 待機→再開、が落ちないこと。**オーブ/林檎での回復は使わない**（捨て垢の有限資源を溶かさない・無料スタミナ範囲で回す方針）。
5. **異常系ハンドリング。** 通信エラー再試行ダイアログ、メンテ画面、再ログイン要求、デイリーログインボーナスのポップアップ等で停止しないこと。未知画面は `advance` タップ＋一定回数で諦めてホーム復帰を試みる、等のフォールバック。

### P2（任意・後回し）
6. **入力高速化。** `adb shell input` は毎回シェル起動で遅い。`uiautomator2`（永続接続）や `minitouch` 系に置き換えると軽くなる。ただし放置ゲーなので必須ではない。
7. **稼働ログ強化。** 周回数に加え、可能ならランク/獲得経験値の推移を記録。長時間の安定性確認に使う。
8. **効率化（任意）。** 学びの力＋ボーナスステージ狙いで経験値効率を上げる構成。ただし安定周回が先。

## 6. やってほしくないこと（NG）

- root化・APK改造・メモリ書き換え・パケット改ざん（検出/規約違反の度合いが跳ね上がる）。
- エミュレータ前提への切り替え。
- 入力を機械的にしすぎる最適化（一定間隔ピッタリ連打など）。遅延・ジッターのランダム性は残す。
- 課金アイテム（オーブ/林檎）を自動消費する処理。

## 7. 受け入れ条件（完成の定義）

- 無人で数時間以上、周回→クリア→もう一度 のループが継続する。
- スタミナ切れで停止せず、待機→自動再開できる。
- 想定外の画面で無限ループ/クラッシュせず、ログに状況が残る。

## 8. 動作確認手順

1. `python monst_rank_bot.py shot` → `screen.png` 確認、テンプレ作成。
2. CONFIG調整後 `python monst_rank_bot.py run`。
3. ログを見て、止まった/誤爆した画面に対応するテンプレ・座標・分岐を追加。
4. 1〜2周ずつ詰めて、最後に長時間放置で安定確認。
